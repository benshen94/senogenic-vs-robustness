# Updating the public paper repository

This document is the standing workflow for taking ongoing private analysis work and updating the public paper repository without losing reproducibility, figure consistency, or data provenance.

## Default prompt to Codex

When the active manuscript, figures, scripts, or results have changed, use a prompt like:

```text
Use $paper-repo-publication. Compare the current manuscript/workspace against the public senogenic-vs-robustness repo, update the repo with the new analysis, figures, fits, cached results, and figure methods docs, then verify from a fresh clone.
```

Include the current manuscript PDF or doc path, the private analysis workspace path, and any figures or result folders that are known to be current.

## Update principles

1. Treat the manuscript as the map of what must be reproducible.
2. Treat scripts and cached result tables as the source of truth for quantitative claims.
3. Keep `analysis/` for runnable paper-specific scripts and `src/` for importable code.
4. Keep `Figures/` as the current manuscript-facing output directory.
5. Keep public data in `data/`; use public summaries instead of raw restricted data when redistribution is not allowed.
6. Keep generated fits, tables, and simulation caches in `results/`.
7. Do not publish private paths, tokens, hidden scratch files, raw restricted data, or local environment state.
8. Every changed non-schematic figure must either be regenerated by a script or have a documented cached source.

## Intake checklist

Before editing the public repo, collect:

- Current manuscript PDF/doc and reporting-summary files.
- Current figure output folder.
- Analysis scripts that generated each changed panel.
- New or changed source data, public summaries, fit records, and cached simulations.
- Notes on expensive simulations: what was rerun, what is cached, and what command would rerun it.
- Any known deviations from the previous manuscript version.

Then inspect:

```bash
git status -sb
git log --oneline -5
find analysis src data results Figures docs scripts -maxdepth 3 -type f | sort
rg -n "(/Users|/home)/|Documents[/]Codex|CloudStorage[/]GoogleDrive|api[_-]?key|access[_-]?token|password|secret" \
  README.md analysis data docs results src scripts pyproject.toml requirements.txt
```

The final `rg` command should return no matches except intentionally documented examples.

## Update loop

1. **Map paper changes.** Read the revised manuscript and list changed claims, figures, panels, methods, datasets, fits, and supplementary outputs.

2. **Map code changes.** Compare the private analysis workspace against the public repo. Identify new scripts, changed helper code, changed cache files, and obsolete files.

3. **Update repo files surgically.** Move or edit only what is needed for the revised paper. Preserve public layout:

```text
analysis/
src/
data/
results/
Figures/
docs/
scripts/
```

4. **Update figure outputs.** Regenerate figure outputs from the repo scripts whenever possible. Keep PDF and PNG outputs. Do not add final SVG artifacts.

5. **Update figure methods docs.** For every changed main or supplementary figure, update the corresponding file under `docs/figure_methods/`. Each doc should describe the panel-level data, scripts, cached inputs, model fits, uncertainty definitions, and reproduction command.

6. **Update README if the public surface changed.** Edit `README.md` only when setup, data, figure commands, repository organization, or citation/licensing notes changed.

7. **Run local verification.**

```bash
rg -n "(/Users|/home)/|Documents[/]Codex|CloudStorage[/]GoogleDrive|api[_-]?key|access[_-]?token|password|secret" \
  README.md analysis data docs results src scripts pyproject.toml requirements.txt
python3 scripts/verify_repo.py
python3 -m compileall -q scripts analysis src
python3 scripts/reproduce_figures.py --set all
```

8. **Compare regenerated outputs to committed outputs.** The target is:

- PNG outputs are byte-identical after rerun, or the intentional visual changes are reviewed and committed.
- PDF outputs render visually identically when only metadata/object-stream bytes change.
- If a figure intentionally changed, document why in the commit message or methods doc.
- `results/index/outputs.csv` should not change merely because scripts were rerun.

9. **Fresh-clone verification.** After committing and pushing, test from GitHub, not just the warm working copy:

```bash
tmpdir=$(mktemp -d /tmp/svr-confirm-clone.XXXXXX)
git clone https://github.com/benshen94/senogenic-vs-robustness.git "$tmpdir/senogenic-vs-robustness"
cd "$tmpdir/senogenic-vs-robustness"
python3 -m venv .venv
source .venv/bin/activate
python3 -m pip install -r requirements.txt
rg -n "(/Users|/home)/|Documents[/]Codex|CloudStorage[/]GoogleDrive|api[_-]?key|access[_-]?token|password|secret" \
  README.md analysis data docs results src scripts pyproject.toml requirements.txt
python3 scripts/verify_repo.py
python3 -m compileall -q scripts analysis src
python3 scripts/reproduce_figures.py --set all
git status --short
```

10. **Clean up.** Remove temporary clones, scratch render comparisons, and pip logs before reporting completion.

## Figure-output comparison standard

Use byte equality for files where it is realistic:

- PNG files should normally be byte-stable after `scripts/reproduce_figures.py --set all`.
- CSV/source tables should be unchanged unless the analysis changed.
- PDF files may differ by Matplotlib PDF serialization, xref offsets, or metadata. In that case, render old and new PDFs and require pixel-identical images at a fixed density.

Example PDF render comparison:

```bash
magick -density 150 "old.pdf[0]" -alpha remove old.png
magick -density 150 "new.pdf[0]" -alpha remove new.png
```

Then compare pixel arrays with Python/Pillow and require `mse == 0`, `max_abs == 0`, and `changed_pixel_frac == 0`.

## Commit standard

Use small, interpretable commits:

- `Update Fig 3 exposure projection`
- `Refresh progeria fit methods docs`
- `Add NHANES exposure survival summaries`
- `Update reproducibility checks for revised figures`

Each commit should say whether figures were regenerated, what inputs changed, and what verification passed.

## When to stop and ask

Stop before publishing if:

- A changed dataset may not be redistributable.
- A figure cannot be regenerated and no cached source table explains it.
- The manuscript claim and script output disagree.
- Fresh-clone installation fails because a dependency is unavailable or platform-specific.
- Regenerated figures differ visually from committed outputs and the difference was not requested.
